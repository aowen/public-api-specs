A Technical Guide for Integrating Scoir and College Application Platforms

Version 0.0.1

March 17, 2022

Overview

Scoir supports integrations with college Application Platforms so students can complete college applications and counselors can monitor status of their students’ applications, all from within the Scoir experience.

Audience and Purpose

This guide is intended to drive technical conversations between Scoir and Application Platforms to achieve specificity around each of the necessary integration points outlined in subsequent sections.

High-Level Integration Flow

  • When a Scoir student starts a college application, Scoir will send the student’s Application Profile to the Application Platform in order to pre-fill answers to standard questions
  • The Application Platform will notify Scoir when it has processed the Application Profile so the student can take subsequent steps toward submitting the application
  • A Scoir student will single sign-on (SSO) into the Application Platform to review and submit their college application, including answering institution-specific questions and payment of any fees
  • The Application Platform will notify Scoir when the application has been submitted

Frequently Asked Questions

Initiating Applications

Q: What are Scoir’s expectations of the Application Platform’s “initiate” API?

A: The API should accept and process a single Scoir Student’s Application Profile in near “real time”. Scoir expects that a student can

initiate, complete and submit an application in a single interactive session.

Q: Does Scoir have an API authentication preference?

A: No, the Application Platform determines authentication requirements. Scoir will support any reasonable authentication mechanisms

used to secure the API (i.e. API key, PKI, etc.).

Q: Does Scoir have an API design preference?

A: Yes, Scoir prefers REST/JSON APIs.

Q: What happens when a student updates their profile on Scoir after submitting an application?

A: Scoir intends to report such changes to the Application Platform in near “real time”. It is up to the Application Platform to determine

whether Scoir calls the “initiate” API endpoint again or a separate API endpoint intended solely for reporting changes to a student’s

Application Profile.

Reporting Application Status

Q: When should the Application Platform call the status API?

A: Scoir wants to provide a near “real-time” experience for students and counselors, so the expectation is that the Application Platform

will report changes in application status as soon as they occur.

Q: What information does the Application Platform need to send to Scoir each time a status change occurs?

Student Application Submission

Q: Must the Application Platform support the ability for a student to SSO into the Application Platform?

A: Yes. Scoir doesn’t currently provide a way for students to answer institution-specific questions or submit application payments and

expects that the student will be able to SSO into the application platform, from Scoir, to complete these tasks.

Q: Does Scoir have a preferred method for asserting user details in support of SSO?

A: Yes, Scoir prefers using JSON Web Tokens (JWT)1 .

Q: Who determines the encryption scheme for the JWT SSO tokens?

A: The Application Platform determines encryption requirements. Scoir will support any reasonable encryption approach using

symmetric or asymmetric cryptography

Q: Can Scoir suggest resources to learn more about how JWT SSO works?

A: Yes, Zendesk has published a great article about Enabling JWT single sign-on.
